# SF3D MTL Generation - Complete Textured 3D Objects

This enhanced version of SF3D now supports **comprehensive mesh export** with both GLB and OBJ+MTL formats, solving the grey mesh issue by generating proper material files.

## 🆕 New Features

### ✅ **MTL File Generation**
- Generates proper Wavefront `.mtl` material files
- Includes texture references (diffuse, normal maps)
- PBR material parameters (roughness, metallic)
- Compatible with Blender, MeshLab, Unity, etc.

### ✅ **Multiple Export Formats**
- **GLB**: Embedded textures (recommended for web/viewing)
- **OBJ + MTL**: Standard format with separate material file
- **Both formats**: Generate simultaneously for maximum compatibility

### ✅ **Enhanced Material Support**
- UV-unwrapped meshes with proper texture coordinates
- PBR materials with roughness/metallic parameters
- Normal map generation and export
- High-resolution texture baking (up to 4096px)

## 🚀 Quick Start

### 1. Activate Environment
```bash
conda activate stable-ai
```

### 2. Basic Usage

#### GLB Export (Default)
```bash
python run.py bicycle.jpg --output-dir outputs --device cuda
```

#### OBJ + MTL Export
```bash
python run.py bicycle.jpg --export-formats obj --generate-mtl --output-dir outputs --device cuda
```

#### Both Formats
```bash
python run.py bicycle.jpg --export-formats glb obj --generate-mtl --output-dir outputs --device cuda
```

### 3. High-Quality Textures
```bash
python run.py bicycle.jpg \
  --export-formats obj \
  --generate-mtl \
  --texture-resolution 4096 \
  --output-dir outputs \
  --device cuda
```

### 4. AI-Enhanced Textures
```bash
python run.py bicycle.jpg \
  --export-formats glb obj \
  --generate-mtl \
  --enable-texture-transfer \
  --reference-image texture_reference.jpg \
  --texture-resolution 2048 \
  --output-dir outputs \
  --device cuda
```

## 📁 Output Structure

```
outputs/
├── 0/
│   ├── mesh.glb                    # GLB with embedded textures
│   ├── mesh.obj                    # Wavefront OBJ mesh
│   ├── mesh.mtl                    # Material file
│   ├── mesh_texture.png            # Diffuse/albedo texture
│   ├── mesh_texture_normal.png     # Normal map (if generated)
│   ├── mesh_ai_textured.glb        # AI-enhanced version
│   └── input.png                   # Background-removed input
```

## 🎯 MTL File Features

The generated `.mtl` file includes:

```mtl
# Material file generated by SF3D
newmtl sf3d_material
Ka 1.000 1.000 1.000    # Ambient color
Kd 1.000 1.000 1.000    # Diffuse color
Ks 0.500 0.500 0.500    # Specular color
Ns 32.000                # Shininess
d 1.000                  # Opacity
illum 2                  # Illumination model

# Texture maps
map_Kd mesh_texture.png           # Diffuse texture
map_Bump mesh_texture_normal.png   # Normal map
```

## 🔧 Command Line Options

| Option | Description | Default |
|--------|-------------|---------|
| `--export-formats` | Export formats: `glb`, `obj` | `["glb"]` |
| `--generate-mtl` | Force MTL generation with OBJ | `False` |
| `--texture-resolution` | Texture atlas resolution | `2048` |
| `--enable-texture-transfer` | AI texture enhancement | `False` |
| `--reference-image` | Reference for AI texture | `None` |
| `--direct-color-mode` | Exact color matching | `False` |

## 🎨 Viewer Compatibility

### ✅ **Blender**
1. `File > Import > Wavefront (.obj)`
2. Select `mesh.obj`
3. Switch to **Material Preview** mode
4. Check **Material Properties > Principled BSDF**
5. Textures should load automatically

### ✅ **MeshLab**
1. `File > Import Mesh`
2. Select `mesh.obj`
3. `Render > Texture > Per-Face`
4. Or: `Render > Texture > Per-Vertex`

### ✅ **Unity/Unreal**
1. Import both `.obj` and `.mtl` files
2. Engine should auto-link materials
3. Manually assign textures if needed

### ✅ **Online Viewers**
- **GLB**: Works with most online 3D viewers
- **OBJ+MTL**: Compatible with viewers that support Wavefront format

## 🔍 Troubleshooting

### Grey Mesh Issues
1. **Check file paths**: Ensure `.obj`, `.mtl`, and texture files are in same folder
2. **Viewer settings**: Switch to Material Preview/Rendered mode
3. **Material linking**: Manually connect textures in material editor
4. **File validation**: Run test script to verify exports

### Missing Textures
```bash
# Validate exported files
python -c "from mesh_export_utils import validate_exported_files; validate_exported_files({'obj': 'outputs/0/mesh.obj', 'mtl': 'outputs/0/mesh.mtl'})"
```

### Memory Issues
- Reduce `--texture-resolution` to 1024 or 512
- Use `--target-vertex-count 30000` for simpler meshes
- Close other applications using GPU memory

```bash
obj_path = '{}'

```
